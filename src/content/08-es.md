# 搜索引擎 (Elasticsearch)

## 核心概念

### 什么是 Elasticsearch？

Elasticsearch（简称 ES）是一个基于 Apache Lucene 的分布式搜索和分析引擎，提供全文搜索、结构化搜索、分析等功能。它具有近实时搜索、高扩展性、RESTful API 等特点。

### 核心概念对照

| ES 概念 | 关系型数据库类比 | 说明 |
|---------|-----------------|------|
| Index | Database | 索引（文档集合） |
| Type | Table | 类型（ES 7.x 后已废弃） |
| Document | Row | 文档（JSON 格式） |
| Field | Column | 字段 |
| Mapping | Schema | 映射（定义字段类型） |
| Shard | Partition | 分片（水平拆分） |
| Replica | Replica | 副本（高可用） |

### ES vs MySQL 查询对比

| 需求 | MySQL | Elasticsearch |
|------|-------|---------------|
| 精确匹配 | `WHERE name = 'Alice'` | `{"term": {"name": "Alice"}}` |
| 模糊匹配 | `WHERE name LIKE '%lice%'` | `{"match": {"name": "lice"}}` |
| 范围查询 | `WHERE age BETWEEN 20 AND 30` | `{"range": {"age": {"gte": 20, "lte": 30}}}` |
| 全文搜索 | `LIKE '%keyword%'`（慢） | `{"match": {"content": "keyword"}}`（快） |
| 聚合统计 | `GROUP BY / COUNT` | `{"aggs": {"group": {"terms": {"field": "category"}}}}` |
| 多条件 | `WHERE a=1 AND b=2` | `{"bool": {"must": [{"term": a}, {"term": b}]}}` |

## Node.js 实现

### 安装与连接

```bash
npm install @elastic/elasticsearch
```

```javascript
const { Client } = require('@elastic/elasticsearch')

const client = new Client({
  node: 'http://localhost:9200',
})

async function checkConnection() {
  const health = await client.cluster.health()
  console.log(`ES Status: ${health.status}`)
}
```

### 索引操作

```javascript
await client.indices.create({
  index: 'articles',
  body: {
    mappings: {
      properties: {
        title: { type: 'text', analyzer: 'ik_max_word' },
        content: { type: 'text', analyzer: 'ik_max_word' },
        author: { type: 'keyword' },
        tags: { type: 'keyword' },
        created_at: { type: 'date' },
        views: { type: 'integer' },
      },
    },
  },
})

await client.indices.delete({ index: 'articles' })
```

### 文档 CRUD

```javascript
await client.index({
  index: 'articles',
  id: '1',
  body: {
    title: 'Elasticsearch 入门教程',
    content: 'Elasticsearch 是一个分布式搜索和分析引擎',
    author: 'Alice',
    tags: ['elasticsearch', '搜索'],
    created_at: '2024-01-01',
    views: 100,
  },
})

const body = articles.flatMap((doc) => [
  { index: { _index: 'articles', _id: doc.id } },
  doc,
])
await client.bulk({ body })

const { source } = await client.get({ index: 'articles', id: '1' })

await client.update({
  index: 'articles', id: '1',
  body: { doc: { views: 101 } },
})

await client.delete({ index: 'articles', id: '1' })
```

### 搜索查询

```javascript
const result = await client.search({
  index: 'articles',
  body: {
    query: { match: { content: '分布式搜索引擎' } },
    highlight: { fields: { content: {} } },
  },
})

const result = await client.search({
  index: 'articles',
  body: {
    query: {
      bool: {
        must: [{ match: { title: 'Elasticsearch' } }],
        filter: [{ term: { author: 'Alice' } }],
      },
    },
    sort: [{ created_at: 'desc' }],
    from: 0, size: 10,
  },
})

const result = await client.search({
  index: 'articles',
  body: {
    size: 0,
    aggs: {
      tags: { terms: { field: 'tags' } },
    },
  },
})
```

## Spring Boot 实现

### 依赖配置

```xml
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-data-elasticsearch</artifactId>
</dependency>
```

```properties
spring.elasticsearch.uris=http://localhost:9200
```

### 文档实体

```java
@Document(indexName = "articles")
public class Article {

    @Id
    private String id;

    @Field(type = FieldType.Text, analyzer = "ik_max_word")
    private String title;

    @Field(type = FieldType.Text, analyzer = "ik_max_word")
    private String content;

    @Field(type = FieldType.Keyword)
    private String author;

    @Field(type = FieldType.Keyword)
    private List<String> tags;

    @Field(type = FieldType.Date)
    private LocalDateTime createdAt;

    @Field(type = FieldType.Integer)
    private Integer views;
}
```

### Repository

```java
public interface ArticleRepository extends ElasticsearchRepository<Article, String> {
    List<Article> findByTitle(String title);
    List<Article> findByAuthor(String author);

    @Query("{\"bool\": {\"must\": [{\"match\": {\"title\": \"?0\"}}]}}")
    List<Article> searchByTitle(String keyword);
}
```

### Service 层

```java
@Service
public class ArticleService {

    @Autowired
    private ArticleRepository repository;

    @Autowired
    private ElasticsearchRestTemplate template;

    public Article save(Article article) {
        return repository.save(article);
    }

    public Optional<Article> findById(String id) {
        return repository.findById(id);
    }

    public Page<Article> search(String keyword, int page, int size) {
        NativeSearchQuery query = new NativeSearchQueryBuilder()
            .withQuery(QueryBuilders.matchQuery("content", keyword))
            .withHighlightFields(
                new HighlightBuilder("content").preTags("<em>").postTags("</em>")
            )
            .withPageable(PageRequest.of(page, size))
            .build();

        SearchHits<Article> hits = template.search(query, Article.class);
        List<Article> articles = hits.getSearchHits().stream()
            .map(SearchHit::getContent)
            .collect(Collectors.toList());

        return new PageImpl<>(articles, PageRequest.of(page, size), hits.getTotalHits());
    }

    public Map<String, Long> tagStats() {
        NativeSearchQuery query = new NativeSearchQueryBuilder()
            .addAggregation(AggregationBuilders.terms("tag_counts").field("tags"))
            .build();

        SearchHits<Article> hits = template.search(query, Article.class);
        Terms terms = hits.getAggregations().get("tag_counts");

        Map<String, Long> result = new HashMap<>();
        for (Terms.Bucket bucket : terms.getBuckets()) {
            result.put(bucket.getKeyAsString(), bucket.getDocCount());
        }
        return result;
    }
}
```

## 知识迁移要点

### 相似点

| 概念 | Node.js (@elastic/elasticsearch) | Spring Boot (Spring Data ES) |
|------|----------------------------------|-------------------------------|
| 连接配置 | `new Client({ node: '...' })` | `application.properties` |
| 创建文档 | `client.index()` | `repository.save()` |
| 查询文档 | `client.search()` | `template.search()` / `repository.findAll()` |
| 查询语法 | JSON Query DSL | `QueryBuilders` / `@Query` |
| 分页 | `from` / `size` | `PageRequest.of()` |
| 聚合 | `aggs` 对象 | `AggregationBuilders` |

### 差异点

| 维度 | Node.js | Spring Boot |
|------|---------|-------------|
| API 风格 | Promise 链式调用 | Repository 模式 / Template |
| 查询构建 | 原生 JSON DSL | `QueryBuilders` 封装 |
| 类型映射 | 手动处理 | `@Document` + `@Field` 注解 |
| 分页结果 | `result.hits.hits` | `Page<T>` 对象 |
| 集成度 | 需要手动管理连接 | Spring 自动管理 |

## 小结

- Elasticsearch 适合**全文搜索、日志分析、数据聚合**场景
- Node.js 使用 `@elastic/elasticsearch` 客户端，直接操作 JSON DSL
- Spring Boot 使用 `Spring Data Elasticsearch`，通过 Repository 模式简化操作
- 中文分词需要安装 **IK Analysis** 插件
- ES 擅长搜索但不适合事务性操作，应与 MySQL 配合使用
