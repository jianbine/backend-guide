# 消息队列 (MQ)

## 核心概念

### 什么是消息队列？

消息队列（Message Queue）是一种应用程序之间的通信方法，应用程序通过读写出入队列的消息来通信，而无需专用连接来链接它们。消息传递指的是程序之间通过在消息中发送数据进行通信，而不是通过直接调用彼此来通信。

### 为什么需要消息队列？

| 场景 | 没有 MQ | 有 MQ |
|------|---------|-------|
| 异步处理 | 用户等待所有操作完成 | 立即返回，后台处理 |
| 流量削峰 | 高并发直接压垮服务 | 队列缓冲，平滑处理 |
| 系统解耦 | 服务之间直接依赖 | 通过消息松耦合 |
| 分布式事务 | 强一致性，性能差 | 最终一致性，高性能 |

### 常见消息队列对比

| 特性 | RabbitMQ | Kafka | Redis Streams |
|------|----------|-------|---------------|
| 语言 | Erlang | Scala/Java | C |
| 协议 | AMQP | 自定义协议 | Redis 协议 |
| 吞吐量 | 万级 | 百万级 | 万级 |
| 延迟 | 微秒级 | 毫秒级 | 微秒级 |
| 持久化 | 支持 | 支持 | 支持（AOF/RDB） |
| 消费模式 | 推/拉 | 拉 | 消费者组 |
| 适用场景 | 业务消息、任务队列 | 日志、流处理 | 轻量级队列 |

## Node.js 实现

### BullMQ（基于 Redis）

```javascript
// npm install bullmq

const { Queue, Worker } = require('bullmq')

// 创建队列
const emailQueue = new Queue('email', {
  connection: {
    host: 'localhost',
    port: 6379,
  },
})

// 生产者 - 发送消息
async function sendEmail(to, subject, body) {
  await emailQueue.add('send-email', {
    to,
    subject,
    body,
  }, {
    attempts: 3,
    backoff: 5000,
    removeOnComplete: true,
  })
}

// 消费者 - 处理消息
const emailWorker = new Worker('email', async (job) => {
  const { to, subject, body } = job.data
  console.log(`Sending email to ${to}: ${subject}`)
  await new Promise(resolve => setTimeout(resolve, 1000))
  return { success: true, to }
}, {
  connection: {
    host: 'localhost',
    port: 6379,
  },
})

emailWorker.on('completed', (job) => {
  console.log(`Job ${job.id} completed`)
})

emailWorker.on('failed', (job, err) => {
  console.log(`Job ${job.id} failed: ${err.message}`)
})
```

### RabbitMQ（amqplib）

```javascript
// npm install amqplib

const amqp = require('amqplib')

async function setup() {
  const connection = await amqp.connect('amqp://localhost')
  const channel = await connection.createChannel()

  await channel.assertQueue('task_queue', { durable: true })

  channel.sendToQueue('task_queue', Buffer.from('Hello World'), {
    persistent: true,
  })

  channel.consume('task_queue', (msg) => {
    console.log(`Received: ${msg.content.toString()}`)
    channel.ack(msg)
  }, { noAck: false })
}
```

### 发布/订阅模式

```javascript
channel.publish('logs', '', Buffer.from('info: Hello World'))

channel.assertExchange('logs', 'fanout', { durable: false })
channel.assertQueue('', { exclusive: true })
channel.bindQueue(q.queue, 'logs', '')
channel.consume(q.queue, (msg) => {
  console.log(`[x] ${msg.content.toString()}`)
}, { noAck: true })
```

## Spring Boot 实现

### RabbitMQ + Spring AMQP

```xml
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-amqp</artifactId>
</dependency>
```

```java
@Configuration
public class RabbitMQConfig {

    @Bean
    public Queue emailQueue() {
        return new Queue("email.queue", true);
    }

    @Bean
    public DirectExchange exchange() {
        return new DirectExchange("app.exchange");
    }

    @Bean
    public Binding binding(Queue queue, DirectExchange exchange) {
        return BindingBuilder.bind(queue).to(exchange).with("email.routing.key");
    }

    @Bean
    public MessageConverter jsonMessageConverter() {
        return new Jackson2JsonMessageConverter();
    }
}
```

```java
@Service
public class EmailProducer {

    @Autowired
    private RabbitTemplate rabbitTemplate;

    public void sendEmail(EmailMessage message) {
        rabbitTemplate.convertAndSend(
            "app.exchange",
            "email.routing.key",
            message
        );
    }
}
```

```java
@Component
@RabbitListener(queues = "email.queue")
public class EmailConsumer {

    @RabbitHandler
    public void handleEmail(EmailMessage message) {
        System.out.println("Received email: " + message.getTo());
    }
}
```

### Kafka + Spring Kafka

```xml
<dependency>
  <groupId>org.springframework.kafka</groupId>
  <artifactId>spring-kafka</artifactId>
</dependency>
```

```properties
spring.kafka.bootstrap-servers=localhost:9092
spring.kafka.producer.key-serializer=org.apache.kafka.common.serialization.StringSerializer
spring.kafka.producer.value-serializer=org.springframework.kafka.common.serialization.StringSerializer
spring.kafka.consumer.group-id=my-group
spring.kafka.consumer.auto-offset-reset=earliest
spring.kafka.consumer.key-deserializer=org.apache.kafka.common.serialization.StringDeserializer
spring.kafka.consumer.value-deserializer=org.apache.kafka.common.serialization.StringDeserializer
```

```java
@Service
public class EventProducer {

    @Autowired
    private KafkaTemplate<String, String> kafkaTemplate;

    public void sendEvent(String topic, String key, String value) {
        kafkaTemplate.send(topic, key, value);
    }
}
```

```java
@Component
public class EventConsumer {

    @KafkaListener(topics = "user-events", groupId = "my-group")
    public void handleEvent(ConsumerRecord<String, String> record) {
        System.out.println("Received: key=" + record.key() + ", value=" + record.value());
    }
}
```

## 知识迁移要点

### 相似点

| 概念 | Node.js | Spring Boot |
|------|---------|-------------|
| 连接配置 | `amqp.connect()` | `application.properties` |
| 队列声明 | `channel.assertQueue()` | `@Bean Queue` |
| 发送消息 | `channel.sendToQueue()` | `rabbitTemplate.convertAndSend()` |
| 消费消息 | `channel.consume()` | `@RabbitListener` |
| 序列化 | `JSON.stringify()` | `Jackson2JsonMessageConverter` |
| 重试机制 | `attempts` 选项 | `@RetryableTopic` / `RetryTemplate` |

### 差异点

| 维度 | Node.js | Spring Boot |
|------|---------|-------------|
| 配置方式 | 代码中直接配置 | 声明式（注解 + 配置文件） |
| 消费者注册 | 手动调用 `consume()` | `@RabbitListener` / `@KafkaListener` 自动注册 |
| 错误处理 | 事件回调 `on('failed')` | 异常捕获 + 死信队列 |
| 连接管理 | 手动管理 | Spring 自动管理（连接池） |
| 消息确认 | `channel.ack()` / `nack()` | 自动确认或手动确认配置 |

## 小结

- 消息队列的核心价值：**异步、解耦、削峰**
- Node.js 生态推荐 **BullMQ**（轻量、基于 Redis）或 **amqplib**（RabbitMQ 原生）
- Spring Boot 生态推荐 **Spring AMQP**（RabbitMQ）或 **Spring Kafka**（Kafka）
- Spring Boot 的注解驱动方式（`@RabbitListener`）比 Node.js 的回调方式更简洁
- 选择 MQ 时需考虑：吞吐量、延迟、持久化需求、运维成本
